# Known Issues / Backlog

## Пожелания пользователя по UI (не реализовано)
1. Интерфейс "мыльный" на 2K-мониторе — нужна поддержка высокого DPI (Windows DPI awareness, `ctypes.windll.shcore.SetProcessDpiAwareness`)
2. Более "soft"/приятный дизайн вместо стандартного tkinter — возможно, стоит рассмотреть `ttkbootstrap` или другую тему
3. Кнопка отмены/паузы обработки — сейчас `process_video()` выполняется в потоке без возможности прервать
4. Более подробный журнал с прогресс-барами по каждому шагу, а не только общий прогресс-бар

## Диаризация требует принятия условий сразу на двух моделях Hugging Face
`pyannote/speaker-diarization-3.1` внутри подтягивает ещё одну gated-модель — `pyannote/speaker-diarization-community-1`. Токен должен принадлежать аккаунту, который принял условия доступа на **обеих** страницах:
- https://huggingface.co/pyannote/speaker-diarization-3.1
- https://huggingface.co/pyannote/speaker-diarization-community-1

Без этого диаризация упадёт с `403 Client Error` при попытке скачать `xvec_transform.npz`, но программа не крашится — корректно откатывается на единого спикера ("Человек 1") и всё равно создаёт документ.

## torchcodec не грузится (некритично, только шум в логах)
При импорте `pyannote.audio` в лог сыпется куча `OSError: Could not load this library: libtorchcodec_core*.dll` для версий FFmpeg 4-8 — `torchcodec` не может найти совместимую сборку под установленный `torch==2.13.0+cpu`. Не блокирует работу (pyannote откатывается на альтернативный способ декодирования аудио), но засоряет журнал предупреждениями. Можно попробовать зафиксировать совместимую версию `torchcodec` или подавить предупреждение, если раздражает.

## Whisper недетерминирован на синтетической TTS-речи
При тестировании 2026-07-19 (см. Changelog) один и тот же короткий аудиофайл, сгенерированный Windows SAPI-голосом, дал заметно разные результаты транскрибации между двумя запусками (от чистого текста до сильно искажённого). Похоже на эффект temperature-fallback механизма Whisper на нетипичном для модели "роботизированном" голосе — на реальной человеческой речи такого разброса, скорее всего, не будет, но стоит перепроверить на настоящем видео с голосом человека.

## venv в проекте битый (исправлено при тестировании)
Старый `env/` указывал на несуществующий путь Miniconda. Пересоздан как `venv/` при живом тестировании 2026-07-19, все зависимости из `requirements.txt` установлены и работают.
